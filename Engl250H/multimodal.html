The Making of this Website

On the right you can see one of the original designs for this website. When first desiging this site, I wanted to have the menu bar on the left side, with the content filling the rest of the right side. I liked the idea of a logo in the corner, mainly because I didn't know what else to put there. Originally I had a very simple color scheme, and I had very few options in the menu bar. Also, I had the content slide into the content area using some JavaScript and CSS. I decided against this idea because while it worked for small amounts of content, there is no way that it would work for the amount of content I would be putting in. So I went to a standard one page for each topic and that is what I have stuck with.

Also, as the amount of content increased, I had many iterations of the menu bar style. The one you see in the image worked for a while, but as I added more pages, it extended past the bottom of my window and I looked for a solution that would allow me to have many more pages without expanding past my browser window. I found a way to hide list items and then open them when their parent element was hovered over. And this is what it currently is, and I like this look.

I also changed the color scheme because I didn't really like the white background. I searched online for some good color schemes because I am pretty bad at finding colors that go together and aren't too dark or too light. I found the colors I am currently using, and implemented them. I then changed the colors in the logo because the colors before did not match at all with the current color scheme.

Rhetorical Analysis

I think this website uses many of the communication elements outlined in the ISUComm Student Guide. One technique that is used in this site is layering. In the top layer, there is the content; this includes the text and images that are inside the bordered regions. This is the information that I feel is important, and therefore is on top. Below that is the sky-blue regions. These regions are the containers for the content. This helps point out to the viewer where to find the important information. And below that is the darker blue. This is the background layer, whose main purpose is to look nice and help with the color scheme.

Another device that is used throughout this site is transormations. I have both dynamic transformations - those that will change depending on user interactions - and static transformations - those are the similar objects that are different sizes. The static transformations include the rectangular shapes. These are important because they are meant to encapsulate the content. In order to do this, they will need to be different sizes to adapt for the specific content that they will be holding. The dynamic transformations help catch the user's attention. For example, using the web browser Firefox, hover your mouse over the logo in the top left. This should surround the logo with a shadow that is a bit bigger than the logo. Then go into one of the menus and hover over an item at the lowest level ("Written Baseline"). This should make the same sort of shadow. This helps the user keep track of what element they currently are hovering over. While it may seem like "eye-candy" I think it is important in creating a nice user experience.
